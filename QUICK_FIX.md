# БЫСТРОЕ ИСПРАВЛЕНИЕ ОШИБКИ 400

## Проблема
Браузер кэширует старую версию кода, которая пытается загрузить несуществующие поля `subtasks` и `plan`.

## Решение
1. **Очистите кэш браузера:**
   - Нажмите `Ctrl + Shift + R` (жесткая перезагрузка)
   - Или `Ctrl + F5`
   - Или откройте DevTools (F12) → вкладка Network → поставьте галочку "Disable cache"

2. **Перезапустите dev-сервер:**
   ```bash
   npm run dev
   ```

3. **Проверьте консоль браузера:**
   - Должны появиться сообщения: "Loading tasks..." и "Tasks loaded: X"
   - Если ошибка 400 всё ещё есть, выполните SQL из `supabase_migration.sql`

## После исправления
- Приложение будет работать без ошибок
- Функции ИИ будут сохранять данные локально (до перезагрузки)
- После добавления полей в БД данные будут сохраняться постоянно

## SQL для Supabase (если нужно)
```sql
ALTER TABLE public.tasks 
ADD COLUMN IF NOT EXISTS subtasks text,
ADD COLUMN IF NOT EXISTS plan text;
```
